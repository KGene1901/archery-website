Shoot Your Shot's API (Application Program Interface) aims to assist you in developing sport-related applications, an example of this would be archery website that utilizes the requests shown in this documentation in its web features.

Overview

This documentation provides a comprehensive guide in using the API references for features such as:

  • Scoreboards
  • Image gallery
  • Event finder

This API makes use of only two HTTP Requests - GET and POST, which makes its implementation quite straightforward.

Example API calls are also provided on the right hand tab to demonstrate the expected responses that your application should receive throughout your development stage and after deployment. These requests and responses are available in a variety of languages e.g. Javascript, Python, Ruby and Java.

Status Codes

Generally, responses from the implementation of this API will return a 200 (OK) code to acknowledge the fact that the communication between client and server was successful. However, in the off chance that the server does go down, a 5xx (server error) will be returned and this internal server error should be handled properly in the source code of the application.

http://127.0.0.1:8090/scoreData

Returns logged scores stored on server. These scores along with their respective information are stored as a JSON with the first three keys storing string values and the final key storing an integer value.

* Note: the response will always be the full scoreboard information - some code will have to be generated to handle/manipulate each specific data from the object returned.

http://127.0.0.1:8090/search?keyword=&time=&loc=

Retrieves and returns events available from the API endpoint based on user input. The response data is in the form of a JSON with each array containing:

  • Name of the event
  • Event Date
  • Event Location (exact area/city )
  • Event Location (country )
  • Link to more info about the event

An example of this JSON is as below:

[{"title":###, "date":###, "location":###, "country":###, "url":###}]

The endpoint uses the startsWith() JS comparison method to find matching events with the given user input as the starting letter/word(s). Furthermore, the toLowerCase() function has also been used to improve the versatility of the request handling by matching the inputs with the event names regardless of case size.

Some potential keyword scenarios:

Request Response
A letter/word/words which matches the result(s) JSON with data is returned
A letter/word/words which does not match any result Message ('No events found with given input. Please try again.') is returned for user to try again
Numbers or symbols Message ('No events found with given input. Please try again.') is returned for user to try again

Headers

X-Powered-ByExpress
Content-Typeapplication/json

Params

keyword

*Required value

If left empty, response will return all events stored on the server (or those which satisfy the other parameters)

time

*Required value

Can take one of three values:

  • null
  • past
  • upc

Real system time is used to compare with the event dates available

loc

*Required value

Can take one of three values:

  • null
  • loc
  • int
